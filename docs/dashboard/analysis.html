Fully Working — High Confidence

Read any .uasset file (devices, widgets, actors) SafeFileAccess + UAssetAPI expand_more
All reads go through SafeFileAccess which copies to a temp directory before parsing. Never locks source files, never risks corruption. Our patched NormalExport.cs fixes UE5.4's ScriptSerializationOffset issue, enabling full property extraction from external actors. Handles 130+ properties per device reliably.
List and inspect all devices in any UEFN level DeviceService (375 LOC) expand_more
DeviceService scans __ExternalActors__/ directories, classifies actors as devices vs. props vs. terrain, and extracts all non-default property overrides. Property importance ranking surfaces meaningful config (team assignments, scoring rules, spawn behavior) while collapsing rendering noise.
Audit projects for issues with health scoring AuditService (6 checks) expand_more
Six real audit checks: missing references, orphaned assets, duplicate definitions, misconfigured devices, Verse compilation warnings, and structural integrity. Each finding is severity-coded (critical/high/medium/low) with actionable remediation. Project health score aggregates across all checks.
Snapshot and diff projects at the property level ProjectDiffService expand_more
Takes full project snapshots capturing every asset and property value. Diffs compare two snapshots and report added/removed/modified assets with property-level change details. Enables tracking what changed between editing sessions, catching unintended modifications, and auditing collaborator changes.
Translate 45 Verse compiler errors to plain English VerseErrorTranslator (7 categories) expand_more
Maps cryptic Verse compiler output to human-readable explanations across 7 categories: type mismatches, missing imports, scope errors, lifetime issues, syntax mistakes, API misuse, and deprecation warnings. Each translation includes a suggested fix with code examples. Covers 45 distinct error patterns observed across the 92-project sandbox.
Generate Verse code from 15 templates VerseDeviceGenerator + VerseUIGenerator expand_more
Template-driven code generation for common UEFN patterns: round managers, scoring systems, spawn controllers, elimination trackers, timer displays, HUD widgets, popup dialogs, inventory systems, zone managers, and more. Output is syntactically valid Verse with proper imports, module structure, and Epic API usage.
Write custom Verse with full project context VerseWriterTools expand_more
Assembles relevant context (existing Verse files, device schemas from .digest files, project structure) and generates custom Verse code. Understands module boundaries, device references, and UEFN-specific patterns. Handles verse generation directly with full project awareness rather than delegating to a separate session.
Create widget blueprints (.uasset binary output) WidgetBlueprintBuilder expand_more
Generates valid UMG Widget Blueprint .uasset files from WidgetSpec definitions. Supports 15+ widget types with correct containment rules derived from analysis of 883 real widgets. Handles Canvas, Overlay, VerticalBox, HorizontalBox, StackBox, Grid, and more. Produces files UEFN can open and edit natively.
Extract multi-device systems from reference library SystemExtractor expand_more
Scans the 92-project reference library to identify and extract complete device "systems" — groups of devices that work together (e.g., a capture point = zone + score manager + UI + spawners). Extracts configurations, positions, and wiring as portable JSON recipes that can be imported into new projects via SystemImporter.
Search device encyclopedia with common configs DeviceEncyclopedia (11 device types) expand_more
Curated knowledge base of 11 key UEFN device types with documented property schemas, common configurations, and usage patterns. Searchable by device type, property name, or gameplay purpose. Includes real-world examples from the reference library showing how professional creators configure each device.
3D scene preview of device positions Three.js + CUE4Parse GLB expand_more
Renders device positions in a 3D viewport using Three.js. CUE4Parse exports meshes as GLB format for accurate geometry representation. Orbit, zoom, and select devices in the scene. Position data extracted from external actor files using VectorPropertyData inside StructPropertyData (our UE5.4 fix).
Visual widget editor with roundtrip Fabric.js canvas expand_more
WYSIWYG widget layout editor built on Fabric.js. Drag and drop widget elements, set properties visually, preview the result. Roundtrips to/from WidgetSpec which feeds into WidgetBlueprintBuilder for .uasset output. Supports content-aware zoom, slot alignment, and Z-order management.
Device wiring graph visualization Force-directed graph expand_more
Visualizes device-to-device connections (event bindings) as an interactive force-directed graph. Nodes represent devices, edges represent wiring. Helps creators understand complex multi-device interactions at a glance. Identifies orphaned devices with no connections.
Cross-map system import/export SystemImporter (JSON recipes) expand_more
Import complete device systems from the reference library into your project. Systems are serialized as JSON recipes containing device types, property configurations, relative positions, and wiring connections. Handles Epic-only references cleanly. User asset dependencies are flagged for manual resolution.
File watching with auto-snapshot Rust notify crate expand_more
Native file system watcher implemented in Rust via the notify crate. Monitors project directories for changes and automatically triggers snapshots. Efficient event debouncing prevents snapshot storms during bulk operations. Surfaces changes to the Tauri frontend in real-time.
Browse 92 reference projects LibraryIndexer expand_more
Indexes and catalogs the full 92-project reference library at Z:\UEFN_Resources\mapContent\. Browse by project, search by device type or asset name, inspect configurations from professional creators' maps. Library projects are always read-only with full safety guarantees.

Working but Incomplete — Medium Confidence

AI game designer (6 templates) GameDesigner expand_more
Classifies game mode descriptions (FFA, Teams, Rounds, Elimination, Zone Control, Race) and generates appropriate device manifests and Verse scaffolding. Works for the 6 supported templates but has TODO stubs in advanced verse generation paths. Cannot yet handle hybrid game modes or custom scoring logic.
Project assembler (folder + verse generation) ProjectAssembler expand_more
Creates complete .uefnproject folder structures with correct directory layout, plugin scaffolding, and initial Verse files. Verse generation covers the basics but has gaps in complex scenarios like multi-module projects or custom class hierarchies.
QA logic tracer (5 of 10 passes) LogicTracer expand_more
Traces device wiring logic to validate gameplay flows. 5 passes operational: connectivity check, dead-end detection, circular reference detection, event chain validation, and scope analysis. 5 remaining passes are stubs: runtime simulation, timing analysis, race condition detection, state machine validation, and edge case enumeration.
Actor cloning for placement ActorPlacementService expand_more
Clones existing actor files and modifies transforms to place new instances. Works reliably for simple actors (single-component devices). Breaks on complex hierarchies with nested subobjects, blueprint-specific initialization data, or actors requiring specific import chain construction.
Device wiring execution ModificationService expand_more
Creates binding entries in .uasset files to wire devices together. Generates the correct property structures for event connections. Has been validated structurally but untested with real UEFN projects — the generated bindings may need property ordering adjustments to match what UEFN expects.
Map generation (planning + placement) MapGenerator expand_more
Generates map layouts with biome configurations, device placement plans, and spatial distribution algorithms. Planning phase works well (BiomeConfig, MapLayout, AssetCatalog). Actual placement depends on having a seed level with the required assets already present.

In Development — Bridge Required

Live actor placement in UEFN bridge.place_actor expand_more
Python bridge Phase 1. Spawn any device or actor at specified coordinates inside the running editor. Bypasses the binary file limitation by using UEFN's own Python API to instantiate actors with correct import chains, component initialization, and world registration.
Geometry creation (floors, walls, boxes) bridge.geometry expand_more
Create level geometry primitives via bridge commands. Essential for game modes that need arena structures (box fights, zone wars). Currently impossible offline because mesh actors require engine-side construction that UAssetAPI cannot replicate.
Verse compilation + error capture bridge.verse_compile expand_more
Trigger Verse compilation from outside the editor and capture compiler output. Uses PIE (Play In Editor) as a workaround since no standalone Verse compiler exists. Enables automated compile-test-fix loops.
Live world state export bridge.world_state expand_more
Export the current state of all actors in the running editor. Captures transforms, properties, and wiring that may differ from the on-disk .uasset files. Essential for QA validation and ensuring WellVersed's state matches UEFN's reality.
Device catalog scan (4,698 devices) bridge.catalog expand_more
Scan UEFN's full device palette programmatically. The editor contains 4,698+ device types that are not documented anywhere. Bridge enables runtime introspection of every device class, its properties, and default values. Would massively expand the device encyclopedia.
Stamp save/place and pre-publish validation bridge.stamps + bridge.publish_audit expand_more
Save reusable "stamps" (grouped actor selections) and place them in new locations. Pre-publish validation runs Epic's publishing checks programmatically to catch issues before attempting a Creator Portal submission.
One-call map generation pipeline execute_generation_plan expand_more
End-to-end: take a game mode description, generate the plan, create geometry, place devices, deploy Verse code, compile, and validate — all in a single orchestrated pipeline. Requires all bridge commands working together with the existing planning services.

Remaining Limitations

4 fundamental limitations

✓ RESOLVED: Actor placement, level geometry creation, and Verse compilation are now handled by the Python bridge (114 commands, 23 modules). These were limitations before the bridge was built — they are now fully functional when UEFN is open with the WellVersed bridge installed.

close
Test gameplay without UEFN
No simulation engine exists outside of Fortnite itself. Device interactions, player movement, and game logic can only be tested by entering Play mode in the editor.
Resolution: Future sim_device_proxy pattern for offline validation
close
Access Fortnite-specific runtime classes
Many Fort* prefixed properties and classes are engine-internal. Even via Python, they expose read-only interfaces that prevent programmatic modification of runtime behavior.
Resolution: No workaround — engine limitation
close
Publish maps programmatically
Map publishing requires the Creator Portal web UI. Epic provides no API for automated submission, version management, or metadata updates.
Resolution: No workaround — requires Epic API
close
Hot-reload files while UEFN is running
UEFN locks project files during editing sessions. External modifications to .uasset files while the editor is open cause corruption or are silently ignored.
Resolution: Bridge path bypasses file locks via in-editor API
