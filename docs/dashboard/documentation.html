01 Overview

WellVersed is a professional UEFN (Unreal Editor for Fortnite) project management studio. It provides read-safe inspection, device management, scene previewing, Verse code intelligence, widget blueprint generation, and a reusable device recipe library — packaged as a desktop application that is safe to run alongside an open UEFN editor.

The product follows a dual-mode philosophy: a powerful terminal/developer tool today, and a consumer-facing AI-powered assistant in the future. The dark-first UI aesthetic and Tauri v2 desktop shell reflect this professional positioning.

03 NDJSON Sidecar Protocol

The Tauri host communicates with the .NET CLI process over newline-delimited JSON (NDJSON) on stdin/stdout. Each request is a single JSON line; each response is a single JSON line.

3.1 ForgeBridge (Rust)

app/src-tauri/src/sidecar.rs implements the ForgeBridge struct:

  • Startup — start() runs dotnet build then spawns dotnet run --no-build -- sidecar
  • Ready handshake — waits for a {"id":"ready"} line from the sidecar before sending requests
  • Async call — call(method, params) assigns an auto-incremented request ID, registers a oneshot channel, writes the request line, then awaits the response with a 30-second timeout
  • Response routing — a background thread reads stdout lines, parses SidecarResponse, and routes each response to the matching pending sender
  • Stderr forwarding — a second background thread forwards sidecar stderr to host stderr with a [forge] prefix
  • Cleanup — Drop closes stdin and kills the child process
// Request format:
{"id":"req-42","method":"list-devices","params":{"levelPath":"/..."}}

// Response format (success):
{"id":"req-42","result":{...},"error":null}

// Response format (error):
{"id":"req-42","result":null,"error":{"code":"NOT_FOUND","message":"...","details":null}}

3.2 Sidecar Methods

The .NET sidecar (WellVersed.CLI sidecar subcommand) handles all forge_* Tauri commands. The project manager (SidecarProjectManager) and library manager (SidecarLibraryManager) maintain state across calls within a session. The optional AssetPreviewService (CUE4Parse) is initialised on demand via preview-init.

Sidecar JSON is serialised with WriteIndented=false, CamelCase, WhenWritingNull to keep each response on a single line.

4.2 Safety System

Three layered safeguards protect UEFN project files:

Operation modes (automatic)

ModeWhen activeReadsWrites
ReadOnlyreadOnly: true in config, or Library projectCopy-on-readBlocked
StagedUEFN editor process running, or URC activeCopy-on-readTo .wellversed/staged/
DirectUEFN not running, My Project typeCopy-on-readTo source, with auto-backup

AssetGuard

AssetGuard.CanModify() gates every write operation. It checks: (1) operation mode, (2) PKG_FilterEditorOnly and UsesEventDrivenLoader flags in the .uasset binary header (cooked detection), (3) path allowlist/blocklist from config. Cooked assets are never modified under any circumstances.

Copy-on-read (SafeFileAccess)

All reads use SafeFileAccess which copies the source file to a temp location before handing it to UAssetAPI. This ensures the source file is never locked while UEFN has it open.

Critical rule Never open a .uasset source file directly with UAssetAPI. Always go through SafeFileAccess. Locking a file that UEFN has open can corrupt the project.

External actors only store non-default property overrides. If a property appears in an external actor file, it was explicitly changed from Epic's default. This is why device inspection shows a small, meaningful property set rather than hundreds of defaults.

7.1 UAssetAPI Patch (ScriptSerializationOffset)

UAssetAPI submodule is at lib/UAssetAPI/ with a custom fix for UE5.4 external actor property parsing.

Problem: NormalExport.Read() unconditionally skipped 4 dummy bytes after the export header. In UE5.4 external actors, ScriptSerializationEndOffset > 0, meaning there is no padding at that position — the skip caused the property reader to land at the wrong offset and miss all properties.

Fix: When ScriptSerializationEndOffset > 0, seek directly to the property data region using the offset fields instead of applying the fixed skip. This enables reading all 130+ properties from devices like vending machines.

The fix is in lib/UAssetAPI/UAssetAPI/ExportTypes/NormalExport.cs around line 122. Engine version is always EngineVersion.VER_UE5_4 from UAssetAPI.UnrealTypes.

Known UAssetAPI gotchas
  • SetPropertyData extends ArrayPropertyData — match it first in switch statements
  • MulticastDelegatePropertyData.ToString() can throw — always wrap in try/catch
  • StructPropertyData.StructGUID (capital D, not StructGuid)

08 Device Recipes

Device Recipes (device-recipe-v1) are reusable multi-device patterns. A recipe captures a group of devices with their relative positions, property configurations, and wiring (signal connections).

Defined in app/src/shared/recipes.ts. Managed in the recipeStore.

Schema

interface DeviceRecipe {
  $schema: 'device-recipe-v1'
  id: string
  name: string
  description: string
  category: RecipeCategory
  tags: string[]
  author?: string
  createdAt: string
  sourceProject?: string
  sourceLevel?: string
  devices: RecipeDevice[]     // roles + offsets + property overrides
  wiring: RecipeWiring[]      // signal connections between device roles
}

Recipe categories

gameplay, combat, spawning, triggers, ui, environment, economy, custom

Helper functions

  • generateRecipeId() — base36 timestamp + random suffix
  • computeCenter(positions) — average position of a group
  • toOffsets(positions, center) — convert absolute to relative offsets
  • inferCategory(devices) — heuristic category from device class names
  • validateRecipe(unknown) — type guard that checks schema version, id, devices array

Built-in stock recipes are bundled in app/src/shared/builtin-recipes.ts. The RecipesPage allows browsing, creating (from live level devices), and exporting as JSON.

09 Widget Blueprint Pipeline

WellVersed can generate UEFN Widget Blueprint .uasset files from a visual spec, complete with paired Verse controller code. The pipeline spans both the Rust/React frontend and the .NET Core backend.

Flow

  1. User builds a widget tree in WidgetEditorPage (canvas + layer tree)
  2. Canvas store produces a WidgetSpec JSON (widget-spec-v1)
  3. forgeValidateSpec() → forge_validate_spec Tauri command → validate-spec sidecar method → WidgetSpec.Validate()
  4. forgeBuildUasset() → forge_build_uasset → build-uasset → WidgetBlueprintBuilder.Build()
  5. forgeGenerateVerse() → forge_generate_verse → generate-verse → WidgetSpecVerseGenerator

WidgetBlueprintBuilder

Uses a clone-and-extend approach: starts from a minimal template .uasset that has valid WidgetBlueprint / WidgetBlueprintGeneratedClass / WidgetTree infrastructure. Each widget gets two exports (editor tree + runtime tree). Each slot relationship also gets a pair of exports.

Widget containment rules

app/src/shared/widget-rules.ts encodes UEFN's UMG widget containment constraints, derived from scanning 883 real Widget Blueprints. It defines which widget types can contain which other types, and what slot properties each parent-child combination supports.

Models

  • src/WellVersed.Core/Models/WidgetSpec.cs — C# widget tree spec
  • src/WellVersed.Core/Models/WidgetRules.cs — containment rule validation
  • src/WellVersed.Core/Models/WidgetValidationError.cs — validation error with path and severity

10 Library System

A Library is a read-only reference collection — a directory of UEFN projects used for searching, pattern discovery, and AI context. It is distinct from the active project. Libraries are always opened in ReadOnly mode.

The production sandbox has approximately 92 UEFN projects at Z:\UEFN_Resources\mapContent\map_resources\ and a collection of Verse code packs at Z:\UEFN_Resources\mapContent\code_resources\.

Indexing

LibraryIndexer.BuildIndex(path) uses WellVersedConfig.DiscoverProjects() to find all project roots, then indexes each:

  • Verse files — classes, functions, device references, imports, summary
  • Assets — by class, with file sizes and project provenance
  • Device types — class names with usage counts across projects
  • Standalone Verse code packs (directories with .verse files but no .uefnproject)

The resulting LibraryIndex JSON is persisted alongside the library path and can be queried via search-library-index (full-text) or browsed by type, Verse file, or device config.

Library pages

Seven pages are dedicated to library browsing: LibraryPage, LibraryManagePage, LibraryBrowsePage, LibraryVersePage, LibraryAssetsByTypePage, LibraryDeviceConfigsPage, LibraryMaterialsPage, LibraryWidgetsPage.

11 Scene Preview (Three.js)

ScenePreviewPage.tsx renders the devices in a level as a 3D scene using Three.js with OrbitControls. Each device is placed at its real-world Unreal coordinate extracted from its external actor transform.

Device representation

  • Each device type is assigned a deterministic colour — common types like ItemSpawnerDevice, VendingMachineDevice, PlayerSpawnDevice, etc. have explicit colour assignments
  • Unknown types get a hash-based HSL colour for visual differentiation
  • Hover state (0xffffff) and selected state (0x00ffaa) are visually distinct
  • Click on a device opens the property inspector panel

CUE4Parse integration (optional)

AssetPreviewService can extract textures and mesh data from Fortnite's Content/Paks directory using CUE4Parse + SkiaSharp. Initialise via previewInit(fortnitePath). This is opt-in and requires a local Fortnite installation. The texture is returned as a base64 PNG data URL.

12 Verse Tools

WellVersed provides several Verse-focused features across its pages and services:

VerseFilesPage

Syntax-highlighted viewer for project .verse files. Parses and outlines: classes (name, parent), functions (name, signature), @editable device references, and using imports. Also surfaces any persistable structs inline.

PersistentDataPage

Dedicated view for persistent data structures. The client-side parser finds all := class / := struct declarations in Verse files and identifies those with the persistable modifier. Shows field names, types, defaults, and source lines.

VerseErrorExplainerPage

Paste a Verse compiler error message to get a pattern-matched explanation with cause, fix instructions, and a before/after code example. Covers common errors including missing return paths, type mismatches, failable expression context violations, and more.

VerseReferencePage

Browses the Verse reference library — Epic's Book of Verse chapters and the project's own ReferenceLibraryPath .verse files. Read via forgeReadVerse / forgeListDirectory.

app/src/renderer/lib/zoom.ts applies zoom via CSS transform: scale() on the #root element. This avoids layout breakage that occurs with body.zoom in some browsers. 

import { applyZoom } from '../lib/zoom'
applyZoom(1.25) // 125% zoom — sets width/height to 80% to compensate
