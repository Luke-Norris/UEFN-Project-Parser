1. Getting Started

First steps
What is WellVersed?

WellVersed is an AI-powered development studio for Unreal Editor for Fortnite (UEFN) projects. It combines a desktop application, a .NET intelligence layer, and an optional Python bridge that runs inside UEFN itself to give you deep introspection and generation capabilities for Creative maps.

WellVersed can read and understand your project's binary .uasset files, parse device configurations, visualize wiring graphs, generate Verse code, build UMG widget blueprints, run QA analysis, and — with the Python bridge connected — execute changes live inside UEFN.

System Requirements
RequirementVersionNotes
Operating SystemWindows 10/11UEFN is Windows-only; WellVersed follows suit
.NET SDK8.0+Required for the sidecar intelligence layer
Node.js18+Required for the Tauri frontend build
Rust1.70+Required by Tauri v2 (installed via rustup)
UEFNv40.00+Only needed for the Python bridge; WellVersed works offline without it
Installation
  1. Clone the repository (includes the UAssetAPI submodule):
    git clone --recursive https://github.com/your-org/wellversed.git
    cd wellversed
  2. Build the .NET sidecar (intelligence layer + MCP tools):
    dotnet build FortniteForge.sln
  3. Install frontend dependencies and launch:
    cd app
    npm install
    npm run dev
    This starts the Tauri desktop app with the .NET sidecar automatically.
First Launch

When WellVersed opens for the first time, you will see the sidebar on the left with navigation sections: Studio, Project, Create, Explore, Verse, Analyze, Library, and Settings. The main area shows the Home page with a welcome message and quick-start actions.

Your first step will be adding a UEFN project. Until you do, most pages will show placeholder states prompting you to set up a project.

2. Setting Up Your First Project

Project management
Adding a UEFN Project
  1. Click Projects in the sidebar (or the setup prompt on the Home page).
  2. Click Add Project.
  3. Browse to the folder containing your .uefnproject file. This is your project root — typically something like C:\Users\You\Documents\Fortnite Projects\MyMap.
  4. Choose the project type: My Project or Library.
  5. WellVersed scans the project structure, discovers levels, and indexes assets.
Project Types
TypeReadsWritesUse Case
My Project Full access (copy-on-read) Staged writes when UEFN open; direct with backup when UEFN closed Your active map — the one you're building
Library Full access (copy-on-read) Always read-only Reference projects for browsing, searching, and learning from
Selecting a Level

After adding a project, navigate to the Levels page in the sidebar. You will see all .umap files in your project. Click a level to select it as your active level. Many features — devices, wiring, scene preview, audit — require a selected level to operate.

Safety Model

WellVersed is built with an uncompromising safety model. Your project files are never at risk.

verified_user
Copy-on-Read Every file read goes through SafeFileAccess, which copies the file to a temporary location before opening it. WellVersed never holds a lock on your source files. This means it is safe to run alongside UEFN with your map open.

3. The Python Bridge (UEFN Integration)

114 commands
What It Does

The Python bridge is a WebSocket server that runs inside UEFN via its experimental Python 3.11 scripting support. It exposes 114 commands that WellVersed can call to execute actions directly in UEFN — placing actors, creating geometry, compiling Verse, adjusting lighting, managing materials, and much more.

science
Untested in Production The Python bridge has been built against UEFN's Python scripting API documentation but has not been tested with a real running UEFN instance. All 114 commands are implemented but should be considered experimental. Your mileage may vary with different UEFN versions.
Setup
  1. Enable Python Editor Scripting in UEFN: go to Edit > Project Settings, search for "Python", and enable Python Editor Script Plugin. This is an experimental feature in UEFN.
  2. Copy the bridge folder: Copy the entire bridge/wellversed/ directory from the WellVersed repository into your UEFN project at Content/Python/wellversed/.
  3. Copy the init script: Copy bridge/init_unreal.py to Content/Python/init_unreal.py.
  4. Restart UEFN. The init_unreal.py script runs on startup and initializes the WebSocket listener on port 9220.
  5. Verify connection: In WellVersed, look for a green connection indicator in the status area. If connected, you can execute bridge commands.
Bridge Command Categories
Actors
actors.py
Spawn, move, rotate, scale, delete actors. Clone existing actors. Batch placement with transform arrays.
Geometry
geometry.py
Create BSP brushes, mesh primitives, custom shapes. Boolean operations on geometry.
Materials
materials.py
Create and assign materials, modify parameters, set textures, configure material instances.
Lighting
lighting.py
Place and configure point lights, spot lights, directional lights. Adjust intensity, color, attenuation.
Verse
verse.py
Compile Verse files, check for errors, hot-reload scripts during development.
Environment

Fog, sky, water, time-of-day settings. Post-processing volume configuration.
Splines

Create spline actors, add/modify control points, attach meshes along spline paths.
Organization

Folder management, actor grouping, layer assignment, world outliner organization.
Viewport

Camera control, viewport focus, screenshot capture, editor camera manipulation.

4. Studio (Chat) — AI Generation

Game design pipeline
How to Use

The Studio page (labeled "Chat" in the sidebar) is where you describe what you want to build. Type a game description in natural language and WellVersed generates a complete implementation plan including devices, wiring connections, Verse code, and spatial layouts.

Quick Actions

Below the chat input, you will find quick-action buttons for common game types:

Capture Point Elimination Item Shop Tycoon Analyze HUD Widget Custom Verse
The Generation Pipeline

When you submit a description, WellVersed runs it through a multi-stage pipeline:

  1. Natural language parsing — the GameDesigner module extracts game type, player count, mechanics, and constraints from your description.
  2. Generation plan creation — builds a GenerationPlan with specific device placements, property overrides, and wiring connections.
  3. Verse code generation — produces complete, compilable Verse files for game logic (timers, scoring, round management, etc.).
  4. Spatial layout — calculates positions for all devices relative to a spawn origin.
  5. Review & execute — you review the plan, then either save it or execute it in UEFN via the bridge.
psychology
Honest Disclosure: Not Real AI The ChatPage calls the GameDesigner module, which is a keyword-based template system — not an actual AI language model. It matches your description against known patterns and assembles pre-built templates. This is effective for common game types but does not perform true reasoning. Real Claude API integration (requiring an Anthropic API key) is planned but not yet implemented.
"Execute in UEFN" vs "Save Plan"

After generation completes, you have two options:

  • Execute in UEFN (requires Python bridge) — sends the plan to the bridge for live execution inside UEFN. Actors are placed, properties are set, and Verse files are deployed.
  • Save Plan (always available) — saves the generation plan as JSON + Verse files to disk. You can manually import these or execute them later.
Example
// Input:
"8 player box fight FFA with rounds"

// Output includes:
- 8x Player Spawner devices (arranged in circle)
- 1x Round Settings Device (RoundCount: 5, TimeLimit: 120)
- 1x Elimination Manager (ScoreToWin: 10)
- 1x HUD Message Device (round announcements)
- 1x box_fight_manager.verse (round logic + scoring)
- Wiring: Spawners -> RoundSettings -> EliminationManager

5. Widget Editor

Visual UMG builder
Opening the Widget Editor

Navigate to Create > Widget Editor in the sidebar. The editor opens with three panels: a Widget Browser on the left, the Canvas in the center, and the Properties Panel on the right. A Layers Panel is available for managing the widget hierarchy.

Loading Widgets

The Widget Browser panel lists available widget types. Click any widget to add it to the canvas. You can also load existing widget blueprints from your project using the file browser at the top of the panel.

Editing
  • Drag and drop widgets on the canvas to reposition them
  • Properties Panel (right side) shows all editable properties for the selected widget — size, color, text content, font, opacity, visibility
  • Layers Panel shows the widget tree hierarchy — drag to reorder, click to select
  • Containment rules are enforced automatically based on data from 883 real UEFN widgets — only valid children can be placed inside containers
Exporting
Export .uasset
Binary widget blueprint
Click "Export .uasset" in the toolbar, select an output directory, and WellVersed builds a binary widget blueprint file that UEFN can import directly. Uses UAssetAPI to construct the binary format.
Gen Verse
Verse controller code
Click "Gen Verse" to generate a complete Verse controller class for the widget. The code is copied to your clipboard and includes all data bindings, event handlers, and UI update logic.
warning
Widget Fidelity: ~80% The .uasset roundtrip (build binary widget from visual editor) works for most widget types but is approximately 80% fidelity, not 100%. Anchors are currently hardcoded to TopLeft. Complex nested widgets with custom materials may not export perfectly.

6. Device Inspector & Wiring

Deep introspection
Device Inspector

Navigate to Devices in the sidebar (requires a selected level). WellVersed parses every external actor in the level and classifies them as either Creative Devices or Props/Terrain. Devices are grouped by type — all Spawn Pads together, all Elimination Managers together, etc.

Click any device to see its full property sheet. WellVersed reads 130+ properties from the binary .uasset file using the patched UAssetAPI (with the ScriptSerializationOffset fix for UE5.4 external actors). Properties are ranked by importance — meaningful configuration (team index, score-to-win, time limit) appears first, while rendering noise (collision presets, LOD settings) is collapsed at the bottom.

Device Wiring

The Device Wiring page shows a force-directed graph of all signal connections between devices. Each device is a node, and each wiring connection (Event → Receiver) is a directed edge. This gives you a bird's-eye view of your entire game logic flow.

Blueprint Graph

The Blueprint Graph page shows a pin-based visualization similar to Unreal's Blueprint editor. Each device appears as a node with input pins (events it receives) and output pins (events it sends). Connections are drawn as bezier curves between pins.

speed
Performance Note The Blueprint Graph renders well with typical device counts (10-30 devices). Maps with 50+ devices in a single level may experience slower rendering. The force-directed layout does stabilize but can take a few seconds on complex graphs.

7. DFA Simulator

Device behavior modeling
What It Does

The DFA (Deterministic Finite Automaton) Simulator models each Creative device in your level as a state machine. It reads the device wiring graph and allows you to simulate event propagation — fire an event on one device and watch the signal cascade through connected devices, changing their states.

How to Use
  1. Select a level that has devices with wiring connections.
  2. Navigate to the DFA Simulator (under Analyze in the sidebar).
  3. Click Run Simulation to initialize all devices to their starting states.
  4. Each device appears as a colored node. Wiring connections are directed edges between nodes.
  5. Click any edge to simulate that transition — the source device fires its event, and the receiving device processes it.
  6. Watch state changes cascade through the graph as downstream devices react.
State Colors
Active — device is running
Idle — waiting for input
Disabled — turned off
Triggered — just fired
Completed — finished its cycle
Controls
  • Step Forward / Step Back — navigate through simulation history one transition at a time
  • Fire Event panel — select a device and a specific event to manually trigger a custom simulation
  • Reset — return all devices to their initial states

8. QA Analysis

10 analysis passes
Running an Audit

Navigate to Audit in the sidebar. You can run either a project-level audit (all levels) or a level-specific audit (current selected level). The audit reads all binary assets, analyzes device configurations and wiring, and produces findings with severity codes.

Health Score

Every audit produces a letter grade from A+ through F. The score is calculated from the number and severity of findings. Critical issues weigh heavily; informational notes have minimal impact.

The 10 Analysis Passes
Dead Devices
Pass 1
Detects devices with no wiring connections — neither sending nor receiving any events. These are likely forgotten or misconfigured.
Orphaned Outputs
Pass 2
Finds devices that send events to nowhere — output wires that are not connected to any receiver.
Infinite Loops
Pass 3
Detects circular wiring chains where device A triggers B triggers C triggers A. These cause infinite event loops at runtime.
Timing Conflicts
Pass 4
Identifies devices that fire events with conflicting timing — e.g., two round-start signals that could race.
Unreachable Spawns
Pass 5
Checks spawn points for spatial isolation — spawns that are far from any gameplay area or inside geometry.
Missing Win Conditions
Pass 6
Verifies that the level has at least one path to triggering a game-over or victory state.
Team Configuration
Pass 7
Validates team assignments — spawns per team, balanced device distribution, team index consistency.
Unused Verse
Pass 8
Finds Verse scripts that are not referenced by any device in the level.
Property Anomalies
Pass 9
Detects unusual property values — negative timers, zero-health settings, score-to-win of 0, etc.
Duplicate Wiring
Pass 10
Finds identical wiring connections (same source event, same target) that fire redundant signals.
Publish Checklist

The Publish page provides pre-publish validation: actor budget check (is the map under UEFN's actor limit?), required device verification (does every team have spawns?), and Verse build status (do all scripts compile?).

9. Verse Tools

Generation + analysis
Verse Files Browser

Navigate to Verse > Verse Files to browse all .verse files in your project. Files are displayed with syntax highlighting in a code viewer. You can search across all Verse files and navigate between them.

Verse Error Explainer

Paste a Verse compiler error message into the Verse Error Explainer and get a human-readable explanation of what went wrong, plus a suggested fix. Covers common issues like type mismatches, missing imports, incorrect event signatures, and scope problems.

Verse Reference

The Verse Reference page lets you search all device APIs, properties, events, and functions. Data is parsed from UEFN's .digest files which contain Verse class/property schema definitions.

Verse Intelligence
Anti-Pattern Detection
Code quality
Scans Verse code for common mistakes: polling when events exist, hardcoded player counts, missing null checks on optionals.
Device Binding Generation
Scaffolding
Given a set of devices, generates the Verse @editable declarations and binding boilerplate.
Refactoring Suggestions
Improvement
Analyzes existing Verse code and suggests structural improvements — extracting shared logic, using events instead of polling.
Verse Generation

WellVersed includes 15 base templates and 20 smart templates for generating Verse code. Base templates cover common patterns (timer manager, score tracker, UI controller). Smart templates generate complete game mechanics (see Chapter 12). All generated Verse is syntactically valid for the UEFN Verse compiler.

warning
Compilation Status Smart templates generate syntactically valid Verse, but the generated code has not been compiled in a real UEFN instance. It follows Epic's Verse language spec and patterns from verified reference code, but edge cases may exist. Always test-compile in UEFN before shipping.

10. Project Management

Diff + watch + backup
Project Diff

The Project Diff page lets you take a snapshot of your project state, make changes in UEFN, and then compare to see exactly what changed. This is particularly useful for understanding what UEFN modifies when you move an actor or change a device property.

  1. Take Snapshot — WellVersed records the current state of all .uasset files (hashes + metadata).
  2. Make changes in UEFN — move actors, adjust properties, add devices.
  3. Compare — WellVersed scans again and shows a diff: added files, removed files, modified files with property-level changes.
File Watcher

When enabled, the file watcher monitors your UEFN project directory for changes. When UEFN saves files (after Ctrl+S or auto-save), WellVersed automatically takes a snapshot. This builds a timeline of changes that you can browse through.

Staged Changes

When WellVersed operates in Staged mode (UEFN is running), all modifications are written to .wellversed/staged/ instead of the source directory. The Staged Changes page shows all pending modifications with a diff preview. You review each change and choose to apply (when UEFN is closed) or discard.

Backup System

Every time WellVersed performs a direct modification (in Direct mode, UEFN closed), it automatically creates a backup of the original file in .wellversed/backups/. Backups are timestamped and can be restored from the UI.

11. Reference Library

92+ reference projects
Adding a Library

Navigate to Library > Manage in the sidebar. Click Add Library and browse to a directory containing UEFN projects. WellVersed will scan recursively and discover all projects (folders containing .uefnproject files). Libraries are always read-only.

Browsing Reference Projects

The Library section provides the same browsing capabilities as your active project — assets, devices, Verse files, widget blueprints, materials — but across all projects in the library. This is invaluable for understanding how other creators solved similar problems.

Cross-Project Search

Use Asset Search to search across all library projects simultaneously. Find specific device types, Verse patterns, material configurations, or widget layouts. Results link back to the source project for full context.

System Extraction

The Recipes page detects multi-device patterns in reference projects — groups of devices that work together (e.g., a scoring system with elimination manager + score tracker + HUD display). These patterns can be extracted as "recipes" for importing into your own project.

info
Spatial Clustering System extraction uses spatial clustering to group nearby devices. This works well for typical map layouts but may produce suboptimal groupings on very large maps (devices far apart) or very small maps (everything clustered together).

13. Claude Code MCP Integration

90+ tools
What It Is

WellVersed exposes 90+ tools via the Model Context Protocol (MCP), allowing Claude Code (Anthropic's terminal AI) to interact with your UEFN project programmatically. This is the "AI-native" interface — Claude Code can read assets, generate Verse, run audits, design games, and execute plans using the same intelligence layer that powers the desktop app.

Setup

Add WellVersed to your Claude Code MCP configuration:

// In .mcp.json or Claude Code settings
{
  "mcpServers": {
    "wellversed": {
      "command": "dotnet",
      "args": ["run", "--project", "src/FortniteForge.MCP", "--", "path/to/forge.config.json"]
    }
  }
}

14. Known Issues & Items Needing Work

Honest assessment

WellVersed is in active development. This section is an honest accounting of what is untested, incomplete, or needs improvement. We believe transparency builds trust.

Critical: Untested in Production
Python bridge has NOT been tested with real UEFN. All 114 commands are implemented against UEFN's Python scripting API docs, but none have been executed in a running UEFN instance. Command signatures, object paths, and API availability may differ from documentation.
ChatPage calls GameDesigner, not actual Claude AI. The Studio chat uses a keyword-based template system for plan generation. It is not connected to any language model API. "AI generation" refers to the rule-based pipeline, not neural network inference.
Widget .uasset roundtrip fidelity is approximately 80%. Most widget types export correctly, but complex nested widgets, custom materials, and some edge cases may not round-trip perfectly. Always validate exported widgets in UEFN.
Widget anchors are hardcoded to TopLeft. The widget editor supports only the TopLeft anchor preset. The remaining 12 anchor presets (TopCenter, TopRight, Fill, etc.) are not yet implemented in the binary export.
3D scene preview shows colored boxes for many meshes. Device positions are accurate, but actual mesh rendering falls back to colored placeholder boxes when CUE4Parse cannot resolve the mesh asset. This is common for custom/modded content.
Needs Improvement
Asset browsers show raw class names for some unknown asset types. When WellVersed cannot classify an asset, it displays the internal UE class name (e.g., BlueprintGeneratedClass) instead of a friendly label.
Blueprint graph can be slow with 50+ nodes. The force-directed layout and bezier curve rendering become noticeably laggy on complex levels. This is a rendering optimization issue, not a data issue.
Verse LSP integration exists but is limited. The language server protocol connection works for basic features, but it does not yet power dependency trees, parameter signatures, or go-to-definition across files.
Smart templates generate valid Verse but haven't been compiled in real UEFN. All 20 templates produce syntactically correct code following Epic's Verse spec, but actual compilation verification has not been performed.
Device wiring binary execution is untested. Writing wiring connections back to .uasset files (the binary format) has been implemented but not verified with UEFN loading the modified assets.
System extraction spatial clustering has edge cases. The recipe extraction algorithm may produce suboptimal device groupings on maps that are very large (devices far apart) or very small (everything overlapping).
GameDesigner NL parsing is keyword-based. The natural language parser uses pattern matching and keyword extraction, not true natural language understanding. Ambiguous or unusual phrasings may be misinterpreted.
No actual Claude API integration. Using real AI reasoning in the generation pipeline would require an Anthropic API key. The current system is entirely rule-based.
Future Work
Test Python bridge with real UEFN v40.00+ — validate all 114 commands against a running UEFN instance and fix any API discrepancies.
Dynamic UMG widgets — state changes, animations, and data binding in generated widget blueprints.
Full anchor preset support — all 13 UMG anchor presets instead of just TopLeft.
Verse LSP-powered dependency trees — full cross-file navigation with parameter signatures and type inference.
Real AI integration — Anthropic API connection in the ChatPage for true natural language game design.
Foliage, materials, and spline tools — bridge commands exist but are untested. Need real UEFN validation.
Performance profiling with real projects — validate memory estimates, actor budgets, and rendering cost predictions.
Cross-map asset import — importing assets from library projects into your active project requires complex dependency resolution for user-created content.

15. Architecture Overview

For developers

WellVersed is a multi-layer system. The desktop app communicates with a .NET intelligence sidecar via NDJSON over stdin/stdout. The sidecar optionally connects to a Python bridge running inside UEFN via WebSocket. Here is the full stack:

Tauri v2 + React 19 + TypeScript + Tailwind
UI, state management, canvas editors
dns
Intelligence Sidecar
.NET 8 + UAssetAPI + CUE4Parse + 90+ MCP Tools
Asset parsing, generation, analysis
cable
Python Bridge
Python 3.11 + WebSocket (port 9220) + 114 commands
Live UEFN control channel
sports_esports
UEFN (Unreal Editor for Fortnite)
UE 5.4 + Python Editor Scripting + Verse
The editor itself
Key Technologies
UAssetAPI
Binary .uasset parsing
Open-source library for reading and writing Unreal Engine .uasset files. WellVersed uses a patched version (submodule at lib/UAssetAPI/) with a ScriptSerializationOffset fix for UE5.4 external actor property parsing.
CUE4Parse
Fortnite PAK file reading
Used for reading Fortnite's packaged assets — meshes, textures, materials. Powers the 3D scene preview and asset thumbnails.
Three.js
3D preview rendering
WebGL-based 3D renderer for the scene preview. Renders device positions, actor transforms, and resolved meshes in a navigable viewport.
Fabric.js
Widget canvas editor
HTML5 canvas library powering the widget editor's drag-and-drop interface, selection, grouping, and visual manipulation.
Tauri v2
Desktop framework
Rust-based desktop framework (replaces Electron). Provides native window management, file system access, and WebView2 rendering.
Model Context Protocol
AI tool interface
MCP server SDK (NuGet 0.1.0-preview.9) exposes all WellVersed capabilities as tools for Claude Code and other MCP clients.
